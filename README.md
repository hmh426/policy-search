# 보험 약관 검색 & Q&A

보험 상품 PDF를 **빠르게 검색**하고 **Claude AI로 약관을 해석**해주는 도구.

---

## 주요 기능

| 기능 | 설명 |
|------|------|
| **하이브리드 검색** | 키워드(BM25) + 의미 기반(벡터) 검색을 RRF로 결합 |
| **관련도 필터링** | 무관한 검색 결과 자동 제외 (벡터 거리 + 명사 매칭 기반) |
| **검색어 하이라이트** | 검색 결과에서 검색어를 노란 배경으로 강조 표시 |
| **한국어 특화** | 한국어 SBERT 임베딩 + Kiwi 형태소 분석 (1글자 동사/형용사 노이즈 제거) |
| **카테고리별 키워드** | 보험금, 진단·수술, 면책·감액 등 10개 카테고리 + 사용자 키워드 관리 |
| **내 키워드 관리** | 자주 쓰는 검색어 등록/삭제 (편집 모드 토글) |
| **약관 Q&A** | 관련 약관을 자동으로 찾아 Claude가 해석해서 답변 (현재 비활성화) |
| **상품별 그룹핑** | 파일명 기준 자동 그룹핑, 상품 단위로 검색 범위 지정 |
| **모두 펼치기/접기** | 검색 결과를 한눈에 보거나 접어서 보기 (토글) |
| **중복 인덱싱 방지** | 파일 해시로 이미 처리된 파일 자동 건너뜀 |
| **대용량 PDF 지원** | 2,000+ 페이지 PDF 배치 처리, 진행률 표시 |
| **다크모드 UI** | 눈이 편한 다크 테마, 반응형 레이아웃 |

## 처음 실행하기 (빠른 시작)

> Python이 설치되어 있어야 합니다. (https://www.python.org 에서 다운로드)

### 1단계: 패키지 설치 (최초 1회)

터미널(명령 프롬프트)을 열고, 프로젝트 폴더로 이동한 뒤 아래 명령어를 입력합니다:

```bash
cd V:\CODE\POLIQ\POLICY
pip install -r requirements.txt
```

> 첫 설치 시 시간이 좀 걸릴 수 있습니다. 한 번만 하면 됩니다.

### 2단계: PDF 파일 준비

`보험상품_PDF/` 폴더에 보험 약관 PDF 파일을 넣습니다.

**파일명 규칙 (중요!):**
- `상품명_문서종류.pdf` 형식으로 저장하면 상품별 자동 그룹핑
- 예시:
  - `(무)하나로 누리는 건강보험_계약자용약관.pdf`
  - `(무)하나로 누리는 건강보험_사업방법서.pdf`
  - `(무)하나로 누리는 건강보험_상품요약서.pdf`
- `_` 앞부분이 같으면 같은 상품으로 묶임

앱 실행 후 📁 문서 관리 탭에서 드래그 업로드도 가능.

### 3단계: 앱 실행

**방법 A — `run.bat` 더블클릭 (가장 간단)**

프로젝트 폴더의 `run.bat` 파일을 더블클릭하면 자동으로 실행됩니다.

**방법 B — 터미널에서 직접 실행**

```bash
python -m streamlit run main.py
```

> 실행하면 브라우저가 자동으로 열리며 `http://localhost:8501` 주소로 앱이 표시됩니다.
> 첫 실행 시 한국어 AI 모델(~400MB)을 자동 다운로드하므로 시간이 걸립니다.

### 4단계: 문서 인덱싱 (최초 1회)

1. 앱이 열리면 **📁 문서 관리** 탭 클릭
2. **인덱싱 시작** 버튼 클릭
3. 인덱싱이 완료되면 **🔍 검색** 탭에서 검색 가능

> 인덱싱은 PDF를 분석해서 검색 가능하게 만드는 과정입니다. 최초 1회만 하면 되고, 새 PDF를 추가했을 때만 다시 하면 됩니다.

### 5단계: 종료

터미널에서 `Ctrl + C`를 누르면 앱이 종료됩니다.
다시 시작하려면 3단계부터 반복하면 됩니다.

## 사용 방법

### 처음 사용 시

1. PDF 파일을 `보험상품_PDF/` 폴더에 넣거나, 📁 문서 관리 탭에서 업로드

3. 인덱싱 완료 후 검색 사용 가능

### 🔍 검색 탭

- 검색어 직접 입력 또는 카테고리별 키워드 클릭으로 즉시 검색
- 10개 카테고리: 보험금, 진단·수술·입원, 면책·감액, 갱신·보험료, 해지·환급, 실손·자기부담, 특약·보장, 계약 관리, 민원·분쟁, 자동차보험
- 좌측 사이드바에서 특정 상품만 선택해 검색 범위 지정

**검색어 강조 / 모두 펼치기:**
- 검색 결과 상단에 **검색어 강조** 토글과 **모두 펼치기** 토글이 있습니다.
- 검색어 강조: 결과 텍스트에서 검색 키워드를 노란 배경으로 하이라이트 (기본 켜짐)
- 모두 펼치기: 모든 검색 결과를 펼쳐서 보기 (기본 켜짐)

**내 키워드 관리:**
- ⭐ 내 키워드 탭에서 자주 쓰는 검색어를 등록하고 클릭으로 바로 검색
- ✏️ 버튼을 눌러 편집 모드 진입 → 키워드 추가 / X 버튼으로 삭제
- ✅ 완료 버튼으로 편집 모드 종료

**관련도 필터링:**
- 검색어와 무관한 결과는 자동으로 필터링됩니다.
- 벡터 거리 기준(코사인 거리 0.3 초과)과 명사 매칭을 결합하여 판단
- BM25에서도 1글자 동사/형용사(할, 될, 한 등)는 노이즈로 제외
- 1글자 일반명사(암, 골 등)는 유지됨

**효과적인 검색 팁:**
- 구체적으로 검색할수록 정확: `보험금` → `암 진단금 지급 조건`
- 2글자 이상 단어 권장: `암` 단독보다 `암 진단`, `암 보험금` 조합이 정확
- 자연스러운 문장도 가능: `수술비는 얼마나 나오나요`

### ❓ 검색방법 탭

- 효과적인 검색 방법 상세 안내
- 카테고리별 **실전 검색 예시 33개** (보험금 청구, 진단/수술, 면책/감액, 계약, 보험료/갱신, 해지/환급, 실손, 특약, 민원 등)

## 검색 엔진 상세

### 하이브리드 검색 흐름

1. **BM25 키워드 검색**: Kiwi 형태소 분석으로 토큰화 → BM25Okapi 매칭
2. **벡터 의미 검색**: KR-SBERT 임베딩 → ChromaDB 코사인 유사도
3. **RRF 융합**: 두 검색 결과를 Reciprocal Rank Fusion으로 통합 점수 산출
4. **관련도 필터링**: 무관한 결과 제거
5. **결과 반환**: 점수 높은 순으로 정렬, 1위 = 100% 기준 상대 백분율 표시

### 관련도 필터링 로직

검색 결과의 정확도를 높이기 위해 다음 기준으로 무관한 결과를 제외합니다:

- **벡터 거리 필터**: 코사인 거리 0.3 초과 시 관련 없음으로 판단
- **명사 매칭 필터**: 검색어의 핵심 명사(NNG, NNP)가 결과 텍스트에 실제 포함되는지 확인
- **BM25 점수 필터**: 평균의 50% 미만인 결과 제외
- 벡터 유사도가 높거나 명사가 매칭되는 경우만 결과에 포함

### 형태소 분석 토큰 필터

- **유지**: 2글자 이상 모든 품사, 1글자 일반명사(NNG)·고유명사(NNP)·외래어(SL, SH)
- **제거**: 1글자 동사(VV), 형용사(VA), 대명사(NP), 의존명사(NNB) → "할", "이", "것" 등 노이즈 방지
- 인덱싱 변경 시 **강제 재인덱싱** 필요

## 설정 변경

### 검색 파라미터

`config.py`에서 주요 파라미터 수정 가능:

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `CHUNK_SIZE` | 600 | 청크 크기 (문자 수) |
| `CHUNK_OVERLAP` | 100 | 청크 겹침 (문자 수) |
| `VECTOR_TOP_K` | 20 | 벡터 검색 후보 수 |
| `BM25_TOP_K` | 20 | BM25 검색 후보 수 |
| `FINAL_TOP_K` | 5 | 최종 반환 결과 수 |
| `RRF_K` | 60 | RRF 상수 |

### 관련도 필터 파라미터

`search.py`에서 수정 가능:

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `MAX_VEC_DISTANCE` | 0.3 | 벡터 거리 기준 (0=동일, 2=정반대) |
| BM25 최소 비율 | 평균의 50% | BM25 점수 최소 기준 |

- `MAX_VEC_DISTANCE`를 낮추면 더 엄격 (관련 없는 결과 더 많이 제거)
- `MAX_VEC_DISTANCE`를 높이면 더 관대 (더 많은 결과 표시)

### 검색어 하이라이트

`main.py`의 `_highlight_text()` 함수에서 하이라이트 스타일 수정 가능:

| 설정 | 현재 값 | 설명 |
|------|---------|------|
| 배경색 | `rgba(255,215,0,0.10)` | 하이라이트 배경 투명도 (0.10 = 10%) |
| 글자색 | `inherit` | 원래 글자색 유지 |

- 투명도 값을 높이면 (예: 0.25) 더 진한 하이라이트
- 투명도 값을 낮추면 (예: 0.05) 더 연한 하이라이트

### UI 커스터마이징

`main.py` 상단의 `<style>` 블록에서 UI 스타일 수정 가능:

| CSS 선택자 | 역할 | 현재 값 |
|-----------|------|---------|
| `.stApp` | 전체 배경색 | `#3C3C3C` |
| `.stMainBlockContainer` | 메인 콘텐츠 좌우 여백 | `padding: 2rem`, `max-width: 100%` |
| `section[data-testid="stSidebar"]` | 사이드바 배경색 | `#2B2B2B` |

`.streamlit/config.toml`에서 Streamlit 테마 설정:

| 설정 | 현재 값 | 설명 |
|------|---------|------|
| `base` | `dark` | 기본 테마 (dark/light) |
| `primaryColor` | `#FF4B4B` | 강조 색상 (토글, pills 선택 등) |

## 문제 해결

**모델 로딩이 느린 경우**
최초 1회만 다운로드. 이후는 캐시 사용으로 빠름.

**스캔 PDF (이미지 기반) 텍스트 추출 안 되는 경우**
텍스트 레이어가 없는 스캔본은 지원하지 않음. OCR 변환 후 사용.

**메모리 부족**
`config.py`에서 `CHUNK_SIZE`를 늘리면 청크 수가 줄어 메모리 절약 가능.

**검색 결과가 너무 적거나 없는 경우**
- `search.py`의 `MAX_VEC_DISTANCE` 값을 높여보세요 (기본 0.3 → 0.5)
- 검색어를 더 구체적으로 입력해보세요

**검색 결과에 무관한 내용이 포함되는 경우**
- `search.py`의 `MAX_VEC_DISTANCE` 값을 낮춰보세요 (기본 0.3 → 0.2)
- 문서 관리 탭에서 **강제 재인덱싱** 실행

**`streamlit` 명령어를 찾을 수 없는 경우**
`python -m streamlit run main.py` 또는 `run.bat` 더블클릭으로 실행.
